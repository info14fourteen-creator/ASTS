# Data Outputs

ASTS should give users data in several forms, because different roles need different outputs.

## 1. Tender Card

For manager and owner.

Contains:

- Tender name.
- Customer.
- Initial price.
- Deadline.
- Status.
- AI recommendation.
- Key risks.
- Missing data.
- Responsible people.
- Next actions.

## 2. Decision Card

For participate/reject decision.

Contains:

- Recommendation: participate, reject, manual review.
- Confidence score.
- Arguments for participation.
- Arguments against participation.
- Required specialist checks.
- Link to source documents.
- Decision history.

## 3. Requirements Register

For tender specialist/legal/sales.

Contains:

- Participant requirements.
- Required documents.
- Delivery requirements.
- Contract/security requirements.
- Deadlines.
- Source references.
- Review status.

## 4. Position Table

For procurement.

Contains:

- Position name.
- Quantity.
- Unit.
- Required characteristics.
- OKPD2/category.
- AI confidence.
- Approved/needs review flag.

## 5. Supplier Request

For suppliers.

Outputs:

- Email text.
- Public quote link.
- Printable request.
- XLSX export if needed.

## 6. Quote Comparison

For procurement and finance.

Contains:

- Supplier.
- Price per line.
- Total amount.
- VAT.
- Delivery terms.
- Analogs.
- Attachments.
- Cheapest, recommended, and risky offers.

## 7. Profitability Sheet

For manager/finance/owner.

Contains:

- Tender revenue.
- Purchase cost.
- Logistics.
- Platform deductions.
- Security/financing costs.
- Bonuses.
- Other costs.
- Gross profit.
- Margin percent.
- Scenario comparison.

## 8. Task Board

For the whole team.

Contains:

- Tasks by role.
- Due dates.
- Reminders.
- Escalations.
- Completed/overdue state.

## 9. Exports

MVP exports:

- XLSX tender analysis.
- XLSX quote comparison.
- PDF decision report.
- JSON API.

Later exports:

- Bitrix24 deal.
- AmoCRM deal.
- 1C import file.
- Email digest.

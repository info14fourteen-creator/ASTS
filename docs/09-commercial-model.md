# Commercial Model

## Sellable Unit

Sell access per company, not per individual at first. Tender workflows involve several roles, and per-user pricing can slow adoption.

## Suggested Packages

### Start

For small teams.

- 1 company.
- Up to 3 users.
- Limited number of tenders per month.
- AI tender analysis.
- Supplier request links.
- XLSX/PDF export.

### Team

For active tender departments.

- 5-15 users.
- More tenders per month.
- Supplier directory.
- Quote comparison.
- Telegram reminders.
- Role-based tasks.

### Pro

For agencies and larger companies.

- More users.
- More tender volume.
- CRM integration.
- 1C export/import.
- Custom reports.
- Priority support.

## Pricing Hypothesis

Start with manual sales and pilot pricing instead of public fixed pricing.

Possible pilot offer:

- setup fee for onboarding and process mapping
- monthly subscription
- optional success/support package

## First Sales Message

> We help your tender team analyze tender documentation, collect supplier prices, and calculate participation economics faster, with controlled tasks and deadlines.

## Proof Metrics

Track:

- time from tender import to decision
- number of tenders analyzed per week
- number of supplier requests sent
- quote response rate
- tenders rejected early due to risk/unprofitability
- missed deadline reduction
- manager hours saved
